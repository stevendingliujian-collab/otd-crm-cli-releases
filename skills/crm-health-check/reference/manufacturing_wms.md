# Playbook：manufacturing_wms
# 适用场景：WMS / 仓储管理 / 仓储物流

## 行业特征

- 需求相对聚焦（仓库作业流程）
- 关键决策因素：条码/PDA 硬件、ERP 对接、仓储负责人认可
- 客户常把 WMS 当纯台账，需引导理解作业管控价值
- 现场执行规范化是核心价值点

## 推荐阶段停留周期

| 阶段 | 建议天数 | 超1.2倍 | 超1.5倍 | 超2倍 |
|------|----------|---------|---------|-------|
| 初步接触 | 5–10 天 | 轻度预警 | 中度预警 | 重度预警 |
| 需求确认 | 7–14 天 | 轻度预警 | 中度预警 | 重度预警 |
| 调研/诊断 | 7–21 天 | 轻度预警 | 中度预警 | 重度预警 |
| 方案设计 | 5–14 天 | 轻度预警 | 中度预警 | 重度预警 |
| 报价/商务 | 7–21 天 | 轻度预警 | 中度预警 | 重度预警 |
| 谈判/采购 | 10–30 天 | 轻度预警 | 中度预警 | 重度预警 |
| 待签约 | 5–14 天 | 轻度预警 | 中度预警 | 重度预警 |

## 各阶段关键角色要求

| 阶段 | 至少应识别的角色 |
|------|-----------------|
| 初步接触 | 仓储负责人或窗口人 |
| 需求确认 | 仓储负责人、IT（ERP对接） |
| 调研/诊断 | 仓储负责人、IT、财务（账务一致性） |
| 方案设计 | 仓储负责人、IT、管理层 |
| 报价/商务 | 决策者、IT（硬件采购确认） |
| 谈判/采购 | 拍板人、采购 |

## 关键推进证据

- 已明确仓库业务模式（入库、出库、调拨、盘点流程）
- 已确认条码/标签/PDA 设备要求及现有硬件条件
- 已识别仓储负责人和 IT
- 已明确与 ERP 的单据对接方式及分工边界
- 已确定一期仓库或场景试点

## 常见风险

- 客户把 WMS 当纯台账软件（未识别作业管控价值）
- 未确认现场扫码硬件条件（无 PDA/条码设备）
- 未明确仓库作业规则（无法设计流程）
- 与 ERP 分工边界不清（容易产生重复建设或数据不一致）
- 多仓库场景未明确先从哪个仓库切入

## 最佳实践

### 加分行为
- 已完成仓库现场走访，了解实际作业流程
- 已确认 ERP 对接方式（接口/文件/手动）
- 已先聚焦一期仓库或一个核心流程
- 已识别硬件采购责任方（客户自采/随软件采购）

### 减分行为
- 没走现场就直接方案报价
- 未确认硬件条件就承诺条码扫码功能
- ERP 接口边界不清就开始设计方案
- 一次性要求覆盖所有仓库和流程

## 当前阶段卡住时的推荐动作

### 卡在需求确认
1. 约仓库现场参观，了解入出库、盘点的实际操作方式
2. 明确当前用什么工具管库存（Excel/ERP/手写）
3. 确认是否有条码/PDA 设备或计划采购
4. 了解与 ERP 的对接现状（有没有接口、接口是否稳定）

### 卡在方案设计
1. 先出一期仓库的流程设计图，不要一次出全部仓库
2. 明确 ERP 对接接口清单
3. 确认客户对"账实一致"的优先级（核心诉求）
4. 硬件选型建议一并输出，避免后期扯皮
