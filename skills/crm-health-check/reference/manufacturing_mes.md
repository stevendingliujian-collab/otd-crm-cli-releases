# Playbook：manufacturing_mes
# 适用场景：MES / MOM / 生产管理 / 制造执行系统

## 行业特征

- 决策链通常较长（老板、生产、质量、仓储/计划、IT 共同影响）
- 需求容易发散，范围需要强约束
- 涉及多个使用部门，现场调研不可跳过
- 常见集成约束：ERP、条码/设备、OA、WMS
- 不建议一上来报全厂大而全方案

## 推荐阶段停留周期

| 阶段 | 建议天数 | 超1.2倍 | 超1.5倍 | 超2倍 |
|------|----------|---------|---------|-------|
| 初步接触 | 7–14 天 | 轻度预警 | 中度预警 | 重度预警 |
| 需求确认 | 7–21 天 | 轻度预警 | 中度预警 | 重度预警 |
| 调研/诊断 | 10–30 天 | 轻度预警 | 中度预警 | 重度预警 |
| 方案设计 | 7–20 天 | 轻度预警 | 中度预警 | 重度预警 |
| 报价/商务 | 10–30 天 | 轻度预警 | 中度预警 | 重度预警 |
| 谈判/采购 | 15–45 天 | 轻度预警 | 中度预警 | 重度预警 |
| 待签约 | 7–21 天 | 轻度预警 | 中度预警 | 重度预警 |

## 各阶段关键角色要求

| 阶段 | 至少应识别的角色 |
|------|-----------------|
| 初步接触 | 窗口人、所属使用部门 |
| 需求确认 | 使用负责人、项目推动人 |
| 调研/诊断 | 生产、质量、仓储/计划、IT |
| 方案设计 | 使用负责人、IT、管理层（至少1人） |
| 报价/商务 | 决策者、使用负责人、采购/财务影响者 |
| 谈判/采购 | 拍板人、采购/法务、项目负责人 |

## 最佳实践

### 加分行为
- 已完成现场调研，形成现状问题清单
- 已明确一期试点范围（先聚焦，再扩展）
- 已识别 ERP/设备/条码/OA 集成约束
- 已同步老板与使用部门共同参与方案评审
- 已明确系统边界（哪些做，哪些不做）

### 减分行为
- 只跟窗口人沟通，没有扩展角色
- 没做调研就直接报价
- 一上来给全厂全流程大方案
- 为赢单提前过度承诺定制开发
- 只讲功能，不讲收益/落地路径
- 预计签约日期明显拍脑袋（与实际阶段严重不符）

## 当前阶段卡住时的推荐动作

### 卡在需求确认
1. 推动现场调研或线上深度访谈（不能停留在电话/微信层面）
2. 明确涉及部门，不要只停留在单一接口人
3. 梳理现状问题，形成问题清单（给客户一份，作为后续方案基础）
4. 识别一期切入点（生产报工？质量追溯？库存管理？）

### 卡在方案设计
1. 把方案按"现状问题 → 目标 → 方案 → 收益 → 边界"重构
2. 拉管理层参与方案汇报（不只给使用部门看）
3. 明确是否先试点后扩展，锁定一期范围
4. 先统一范围，再讨论价格

### 卡在报价/商务
1. 不急于降价，先判断预算与拍板逻辑
2. 识别竞品情况和替代方案（自研/其他供应商）
3. 判断客户偏买断还是订阅/SaaS
4. 输出报价逻辑文档，不只给一个数字

### 卡在谈判/采购
1. 确认拍板人是否已真正介入谈判
2. 了解采购/合同流程，提前准备所需材料
3. 识别法务/招采可能提出的阻力，提前准备应对方案
4. 建立签约前置动作清单，明确每一步的责任人和时间节点

## 行业化分析参考

按 manufacturing_mes 项目规律，当前阶段**理想状态**应已完成：

**进入调研/诊断前**：明确使用部门 + 识别主要痛点 + 约定调研计划
**进入方案设计前**：完成现场调研 + 形成问题清单 + 确认一期范围
**进入报价前**：管理层参与方案确认 + 锁定一期范围 + 初步判断预算
**进入谈判前**：正式报价 + 客户接受报价逻辑 + 识别决策流程
